# Resend 이메일 설정 가이드

포트폴리오 사이트의 문의 폼이 실제로 이메일을 전송하려면 Resend API 키가 필요합니다.

## 1. Resend 계정 생성

1. https://resend.com 접속
2. 무료 계정 가입 (GitHub 계정으로 가능)
3. 무료 티어 제한:
   - 월 3,000건 이메일
   - 일 100건 이메일
   - 개인 포트폴리오에는 충분합니다

## 2. API 키 발급

1. Resend 대시보드 접속
2. 좌측 메뉴에서 "API Keys" 클릭
3. "Create API Key" 버튼 클릭
4. 키 이름 입력 (예: "Portfolio Contact Form")
5. Permission: "Sending access" 선택
6. "Create" 클릭
7. **생성된 API 키를 복사** (한 번만 표시되므로 꼭 저장!)

## 3. 환경 변수 설정

1. 프로젝트 루트의 `.env.local` 파일 열기
2. 다음과 같이 API 키 입력:

\`\`\`env
RESEND_API_KEY=re_your_actual_api_key_here
\`\`\`

3. 파일 저장

## 4. 개발 서버 재시작

환경 변수를 적용하려면 개발 서버를 재시작해야 합니다:

\`\`\`bash
# 현재 실행 중인 서버 종료 (Ctrl+C)
# 다시 시작
npm run dev
\`\`\`

## 5. 테스트

1. 브라우저에서 http://localhost:3000 접속
2. Contact 섹션으로 스크롤
3. 문의 폼 작성 후 "메시지 보내기" 클릭
4. `sksyag@naver.com`으로 이메일이 전송됩니다

## 6. 무료 티어 제한 관리

Resend 무료 티어는 다음과 같이 자동으로 제한됩니다:

- 일일 100건 초과 시: 다음 날까지 전송 불가
- 월 3,000건 초과 시: 다음 달까지 전송 불가
- 제한 초과 시 API가 에러를 반환하며, 사용자에게 "이메일 전송에 실패했습니다" 메시지가 표시됩니다

별도의 추가 비용 청구는 없습니다. 무료 티어 한도 내에서만 작동합니다.

## 7. 프로덕션 배포 시

Vercel이나 다른 호스팅 서비스에 배포할 때:

1. 호스팅 서비스의 환경 변수 설정으로 이동
2. `RESEND_API_KEY` 변수 추가
3. API 키 값 입력
4. 재배포

## 주의사항

- `.env.local` 파일은 절대 Git에 커밋하지 마세요 (이미 .gitignore에 포함됨)
- API 키가 설정되지 않으면 폼은 작동하지만 이메일은 전송되지 않습니다
- 답장은 문의자의 이메일로 직접 전송됩니다 (replyTo 설정됨)
